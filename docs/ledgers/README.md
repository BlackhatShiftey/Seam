# SEAM Topic Ledgers

Topic ledgers hold stable facts for high-value routes. They are not session logs
and they do not replace `HISTORY.md`.

Use ledgers for facts that future agents should find quickly:

- current state
- known failures
- maintenance decisions
- verification commands
- source-of-truth files
- next safe action

Full chronology stays in `HISTORY.md`; route structure lives in
`tools/history/routing_manifest.json`.
